# 本地预览网页操作说明

这个仓库是 Quarto 网站。每次修改 `index.qmd`、`publications.qmd`、`styles.css` 等文件后，可以用下面的方法在本地预览效果。

## 2. 启动本地预览

推荐使用固定端口 `4000`：

```bash
~/.local/bin/quarto preview --host 127.0.0.1 --port 4000 --render all
```

如果系统已经能直接找到 `quarto`，也可以用：

```bash
quarto preview --host 127.0.0.1 --port 4000 --render all
```

启动后，在浏览器打开：

```text
http://127.0.0.1:4000/
```

在 macOS 里也可以直接运行：

```bash
open http://127.0.0.1:4000/
```

## 3. 修改后如何刷新

`quarto preview` 会监听文件变化。保存 `.qmd` 或 `styles.css` 后，Quarto 通常会自动重新渲染，浏览器刷新后就能看到新效果。

如果页面没有变化，先在浏览器里强制刷新：

```text
Command + Shift + R
```

如果仍然没有变化，停止预览后重新启动。

## 4. 停止本地预览

在运行 `quarto preview` 的终端窗口里按：

```text
Control + C
```

## 5. 如果 4000 端口被占用

查看是谁占用了端口：

```bash
lsof -nP -iTCP:4000 -sTCP:LISTEN
```

如果只是旧的 Quarto 预览进程，可以结束它：

```bash
pkill -f "quarto preview"
```

然后重新启动：

```bash
~/.local/bin/quarto preview --host 127.0.0.1 --port 4000 --render all
```

也可以临时换一个端口，例如 `4001`：

```bash
~/.local/bin/quarto preview --host 127.0.0.1 --port 4001 --render all
```

然后打开：

```text
http://127.0.0.1:4001/
```

## 6. 只渲染，不打开预览

如果只是想检查网站能不能成功生成：

```bash
~/.local/bin/quarto render
```

生成后的静态网站在：

```text
_site/
```
